## 1. Git (Version Control)

**What it is:** A distributed version control system that tracks changes in any set of computer files, usually used for coordinating work among programmers.

**Key Terminology:**

* **Repository (Repo):** The project folder where Git tracks changes.
* **Commit:** A saved snapshot of your code at a specific point in time.
* **Branch:** An independent line of development (e.g., `main`, `feature-login`).
* **Merge:** Combining changes from one branch into another.
* **Stash:** Temporarily shelving uncommitted changes so you can switch branches without losing work.

**Major Commands:**

```bash
# Initialize a new repository
git init

# Clone an existing repository
git clone <url>

# Check the status of your files (staged, unstaged, untracked)
git status

# Create and switch to a new branch
git checkout -b <branch_name>

# Stage all changed files for a commit
git add .

# Create a commit with a descriptive message
git commit -m "feat: add user authentication"

# Push commits to the remote repository
git push origin <branch_name>

# Fetch and merge changes from the remote repository
git pull origin <branch_name>

# View commit history
git log --oneline

```

## 2. Docker & Docker Compose (Infrastructure)

**What it is:** Docker packages an application and its dependencies into a standardized unit called a container. Docker Compose is a tool for defining and running multi-container Docker applications.

**Key Terminology:**

* **Image:** A read-only template with instructions for creating a Docker container (the blueprint).
* **Container:** A runnable instance of an image.
* **Dockerfile:** A text document containing all the commands a user could call on the command line to assemble an image.
* **Volume:** A mechanism for persisting data generated by and used by Docker containers (crucial for databases).
* **Port Mapping:** Connecting a port on your host machine to a port inside the container (`host_port:container_port`).

**Major Commands (Docker):**

```bash
# Build an image from a Dockerfile in the current directory
docker build -t my-app-name:latest .

# Run a container in detached mode (-d) with port mapping (-p)
docker run -d -p 8000:8000 --name my_running_app my-app-name:latest

# List running containers
docker ps

# List ALL containers (running and stopped)
docker ps -a

# View logs for a specific container (follow the output with -f)
docker logs -f <container_name>

# Execute a command inside a running container (open a bash shell)
docker exec -it <container_name> /bin/bash

# Stop and remove a container
docker stop <container_name>
docker rm <container_name>

```

**Major Commands (Docker Compose):**

```bash
# Start all services defined in docker-compose.yml in the background
docker-compose up -d

# Rebuild images before starting (useful if you changed the Dockerfile or requirements.txt)
docker-compose up --build -d

# Stop and remove containers, networks, and images created by 'up'
docker-compose down

# Stop containers and destroy volumes (WARNING: This deletes database data!)
docker-compose down -v

```

## 3. PostgreSQL (Database)

**What it is:** A powerful, open-source object-relational database system known for reliability, feature robustness, and performance.

**Key Terminology:**

* **Schema:** The logical blueprint or architecture of how data is organized (tables, views, indexes).
* **Table:** A collection of related data held in a structured format within a database.
* **Primary Key:** A unique identifier for each record in a table.
* **Foreign Key:** A field in one table that uniquely identifies a row of another table (creates relationships).
* **Index:** A data structure that improves the speed of data retrieval operations.

**Major Commands (via `psql` terminal):**

```bash
# Access the PostgreSQL terminal
psql -U postgres

# Connect to a specific database
\c my_database

# List all databases
\l

# List all tables in the current database
\dt

# View the schema of a specific table
\d table_name

# Exit psql
\q

```

## 4. SQLAlchemy (Python ORM)

**What it is:** The Python SQL toolkit and Object Relational Mapper (ORM) that gives application developers the full power and flexibility of SQL using Python classes.

**Key Terminology:**

* **Engine:** The starting point for any SQLAlchemy application; it manages connection pools to the database.
* **Session:** Establishes all conversations with the database and represents a "holding zone" for all the objects you've loaded or associated with it during its lifespan.
* **Model:** A Python class that maps to a specific table in the database.
* **Declarative Base:** A base class that maintains a catalog of classes and tables relative to that base.

**Basic Usage:**

```python
from sqlalchemy import create_engine, Column, Integer, String
from sqlalchemy.orm import declarative_base, sessionmaker

# 1. Setup Engine and Session
DATABASE_URL = "postgresql://user:password@localhost/dbname"
engine = create_engine(DATABASE_URL)
SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)

# 2. Define Base and Model
Base = declarative_base()

class User(Base):
    __tablename__ = "users"
    id = Column(Integer, primary_key=True, index=True)
    email = Column(String, unique=True, index=True)

# 3. Usage inside a route/service
db = SessionLocal()
new_user = User(email="test@example.com")
db.add(new_user)
db.commit()
db.refresh(new_user)
db.close()

```

## 5. Alembic (Database Migrations)

**What it is:** A lightweight database migration tool for usage with the SQLAlchemy Database Toolkit for Python.

**Key Terminology:**

* **Migration/Revision:** A script that applies changes (create table, add column) to a database schema.
* **Head:** The most recent migration revision.
* **Upgrade:** Applying migrations to move the database schema forward in time.
* **Downgrade:** Reverting migrations to move the database schema backward in time.

**Major Commands:**

```bash
# Initialize Alembic in your project (creates an 'alembic' folder and alembic.ini)
alembic init alembic

# Autogenerate a migration script based on changes made to your SQLAlchemy models
alembic revision --autogenerate -m "create users table"

# Apply all pending migrations to the database (bring it up to 'head')
alembic upgrade head

# Revert the very last migration applied
alembic downgrade -1

```

## 6. FastAPI (Web Framework)

**What it is:** A modern, fast (high-performance), web framework for building APIs with Python 3.8+ based on standard Python type hints.

**Key Terminology:**

* **Path Operation (Route):** The combination of a URL path (e.g., `/users`) and an HTTP method (e.g., `GET`, `POST`).
* **Dependency Injection:** A system where code declares what it needs to function (dependencies), and the framework automatically provides them (e.g., passing a database session to a route).
* **Pydantic Model:** Classes used to define and validate the shape of data coming into (requests) and going out of (responses) your API.
* **ASGI:** Asynchronous Server Gateway Interface. The standard interface between async-capable Python web servers, frameworks, and applications.
* **Uvicorn:** A lightning-fast ASGI server implementation, used to serve FastAPI applications.

**Major Commands & Usage:**

```bash
# Start the development server with live-reloading
uvicorn main:app --reload

```
